# 催更姬 — 项目技术文档

> **版本**: v0.1.0 | **协议**: AGPL-3.0 | **技术栈**: Express + Electron + Playwright

---

## 一、项目定位

催更姬是一个 **AI 辅助长篇小说创作工具**。作者在编辑器中写作，AI 根据世界观设定、角色信息、前情提要智能续写正文。支持桌面 (Electron) 和 Web 两种使用方式。

**核心差异化价值**：不是通用的 AI 聊天工具，而是深度理解"小说创作工作流"的专业工具——管理章节结构、追踪角色状态、维护世界观一致性、控制上下文预算。

## 三、记忆系统（核心创新）

记忆系统是催更姬区别于通用 AI 工具的关键。它实现了**7 层上下文管线**，每次 AI 调用前自动组装：

```
┌──────────────────────────────────────────────────┐
│  Layer 1  Platform     平台规则 (6%)              │
│           任务描述、输出语言、书名                  │
├──────────────────────────────────────────────────┤
│  Layer 2  Author       作者偏好 (3%)              │
│           写作风格、习惯设定、全局规则              │
│           数据源: author/memory/*.md              │
├──────────────────────────────────────────────────┤
│  Layer 3  WorldSetting 世界设定 (14%)             │
│           世界书条目关键词匹配 + 全局设定注入        │
│           检索: 主关键词 + 次级关键词 (AND/OR逻辑)  │
│           支持 sticky/cooldown/probability 机制   │
├──────────────────────────────────────────────────┤
│  Layer 4  CharacterState 角色状态 (16%)           │
│           角色出场检测 + 角色卡信息注入             │
│           状态推断: 当前场景中 / 已离场 / 待出场    │
├──────────────────────────────────────────────────┤
│  Layer 5  PlotHistory   前情资料 (16%)            │
│           NovelMemory + 大纲进度 + 章节摘要         │
│           数据源: novels/<id>/memory/*.md         │
├──────────────────────────────────────────────────┤
│  Layer 6  RecentPlot    近期情节 (24%)            │
│           当前章节正文 + 章节标题                   │
├──────────────────────────────────────────────────┤
│  Layer 7  UserMessage   用户输入 (21%)            │
│           聊天框消息 / 续写请求                     │
└──────────────────────────────────────────────────┘
```

**预算分配**: 内存预算由用户可调（轻量3% → 尽量多35%），L3+L4+L5 按比例缩放。

### 世界书引擎 (WorldBookEngine)

兼容 SillyTavern 世界书格式 (`.json`)。核心检索逻辑：

```
正文扫描 → 主关键词匹配 → 次级关键词逻辑判定 → 概率检查 → 条目激活
```

- **主关键词**: 在扫描深度内匹配到⇒进入下一步
- **次级关键词**: AND_ANY(任中一个) / NOT_ALL(不全中) / NOT_ANY(全不中) / AND_ALL(全中)
- **概率**: 100=必定激活，30=三成概率（增加变化）
- **位置**: 0=角色前 / 1=角色后 / 2=按深度 / 3=@D标注

### 角色出场检测

扫描当前正文，匹配角色名 ⇒ 注入角色卡信息（描述/性格/背景/内嵌世界书）。

### 记忆自动提取 (L3)

AI 生成后自动分析输出文本：
- 检测新角色名（中文姓名模式 + 后缀匹配）
- 检测世界观元素（地点/装备/功法/势力）
- 生成摘要并保存到项目记忆目录

## 五、AI 多模型适配

支持 20+ 模型服务商，统一接口调度：

| 类别 | 服务商 |
|------|--------|
| 海外主流 | Anthropic Claude / OpenAI / Google Gemini / Mistral / xAI Grok / Groq / OpenRouter |
| 国内 | DeepSeek / 通义千问 / 豆包 / 讯飞星火 / GLM / Kimi / SiliconFlow / MiniMax |
| 本地 | Ollama / 自定义 OpenAI 兼容 API |

**ai-client.js** 处理:
- 各服务商消息格式适配
- Token/上下文限制探测 (GET /models/{id})
- 模型列表获取与缓存
- 请求超时控制 (60s abort)

## 六、AI 提取设定功能 (NEW)

`POST /api/ai/extract` — 将已有小说正文导入 AI，自动提取：

**角色提取** (8个维度):
- 姓名、外貌（白描去比喻）、身份、性格线索、关键事件、关系、能力、出场章节

**世界观提取** (A/B/C 类型判定):
- 地点（城市/建筑）、势力（组织/门派）、规则（魔法体系/社会法则）、物品（道具/武器）

**前端交互**:
- AI 面板「提取设定」按钮
- 结果弹窗展示提取的角色+世界观
- 勾选后选择性导入到角色库和世界书

## 八、Prompt 模板系统

兼容 SillyTavern 预设格式，支持导入/导出 `.json`。

**模板编辑器** (二级弹窗):
- 左侧导航：模板列表（启/禁开关 + 批量模式）+ 可拖动分隔条
- 右侧表单：名称、优先级(高/中/低)、系统提示词/占位标记、内容编辑
- 批量操作：全选/反选 + 启用/停用/删除

**注入机制**：模板按优先级分层注入到 System Prompt 中。

## 九、技术要点

| 特性 | 实现 |
|------|------|
| Token 估算 | 中文 ~1.5 chars/token, 英文 ~3.5 chars/token |
| 上下文裁剪 | 从开头截断，保留最近内容，在段落边界切割 |
| 模型上下文探测 | 获取模型 API → 读取 context_length/max_context_length → Hostname 回退 |
| 自动保存 | debounce 延迟(1/2/5秒可配) + beforeunload keepalive 紧急保存 |
| 离线化 | 纯本地字体栈，无外部 CDN 依赖 |
| 全局 Tooltip | body 级 fixed 定位，z-index 最大值，不受任何 overflow 裁剪 |
| Toast 通知 | 左下角弹出动画，自动消失，支持 success/error/warn/info/loading |
